# Plan: 3D Ağız Simülasyonu

## Context

DentAI şu an öğrencilere klinik vakaları 2D JPEG görsellerle sunuyor. Etkileşimli bir 3D ağız/diş modeli, öğrencinin lezyonu döndürüp inceleyebilmesini sağlar — tıp simülatörlerindeki gibi. İstek: mevcut vaka akışına entegre, öğrencinin üzerinde çalışabileceği 3D bir ağız simülasyonu.

---

## Zorluk Değerlendirmesi

### Asıl Sorun: 3D Model Kaynağı

3D görüntüleme kodu yazmak kolaydır. **Asıl zorluk klinik olarak doğru 3D modeller bulmak/üretmektir:**

| Yaklaşım | Zorluk | Süre | Gerçekçilik |
|----------|--------|------|-------------|
| A) Hazır anatomik .glb modeli + lezyon overlay | ★★★☆☆ | 2-3 hafta | Orta |
| B) Gerçek hasta CBCT/intraoral scanner verisi | ★★★★★ | Aylar + etik izin | Yüksek |
| C) Lesyon annotasyonlu basit ağız şeması (2.5D) | ★★☆☆☆ | 1 hafta | Düşük |

**Önerilen: Yaklaşım A** — Sketchfab/Poly Haven gibi kaynaklardan ücretsiz lisanslı dental anatomik .glb modeli bulunur, lezyon bölgeleri highlight olarak eklenir.

---

## Önerilen Yaklaşım: React Three Fiber + GLB Model

### Teknik Stack (Mevcut Sisteme Eklentiler)

```
frontend/package.json additions:
  - three ^0.160.0
  - @react-three/fiber ^8.17.0
  - @react-three/drei ^9.99.0   (OrbitControls, useGLTF, HTML overlays)
```

### Yeni Bileşenler

```
frontend/components/OralSimulator/
  ├── OralSimulator.tsx          ← Ana 3D canvas + OrbitControls
  ├── LesionHighlight.tsx        ← Lezyon marker (mesh overlay veya HTML sprite)
  ├── ToothMap.tsx               ← FDI notasyonu ile diş haritası
  └── useOralModel.ts            ← GLB yükleme + reveal state hook
```

### Vaka Verisiyle Entegrasyon

`case_scenarios.json`'a yeni alan eklenir:

```json
"oral_model": {
  "model_file": "assets/models/oral_cavity_base.glb",
  "lesion_regions": [
    {
      "region_id": "bukkal_mukoza_sag",
      "label": "Sağ bukkal mukoza",
      "highlight_color": "#ff4444",
      "reveal_on": "perform_oral_exam"
    }
  ]
}
```

Backend `cases.py`'ye yeni endpoint:
```
GET /api/cases/{case_id}/model/{filename}   (JWT korumalı, FileResponse .glb)
```

`assets/models/` klasörüne `.gitignore`'dan çıkarılacak yeni dizin eklenir.

### Frontend Entegrasyon Noktası

`ClinicalImagePanel.tsx` yerine veya yanına, `chat/[case_id]/page.tsx` içinde tab/panel olarak eklenir:

```
[Klinik Görsel] [3D Model] ← tab geçişi
```

`revealedMedia` state'i genişletilir: image reveal ile aynı mekanizma (`reveal_on` action tetikler), 3D lezyon highlight da aynı `beklenen_eylemler` sistemine bağlanır.

## Süre Tahmini

| Görev | Süre |
|-------|------|
| three.js/R3F kurulum + temel OrbitControls viewer | 2-4 saat |
| Ücretsiz dental .glb model bulma ve temizleme | 1-2 gün |
| LesionHighlight bileşeni (mesh overlay) | 1 gün |
| Backend model endpoint + JWT koruması | 2-4 saat |
| case_scenarios.json şema güncellemesi (12 vaka) | 1 gün |
| chat sayfasına tab entegrasyonu | 4-6 saat |
| **Toplam (1 geliştirici)** | **~1 hafta** |

**Uyarı:** Kaliteli ücretsiz dental GLB model bulunamazsa (Sketchfab lisansları değişken) model temini haftalara uzayabilir. Bu tek risk faktörüdür.

## Kritik Dosyalar

**Değiştirilecek:**
- `frontend/package.json` — three.js bağımlılıkları
- `frontend/app/chat/[case_id]/page.tsx` — tab paneli ekle
- `backend/app/api/routers/cases.py` — `/model/{filename}` endpoint
- `backend/data/case_scenarios.json` — `oral_model` alanı (12 vakaya)
- `.gitignore` — `assets/models/` izin ver

**Yeni oluşturulacak:**
- `frontend/components/OralSimulator/` (4 dosya)
- `assets/models/` dizini + base GLB model

**Yeniden kullanılacak:**
- `ClinicalImagePanel.tsx`'teki `useJwtImage` hook pattern → model fetch için aynı JWT mekanizması
- `cases.py`'deki `_ASSETS_IMAGES_DIR` pattern → `_ASSETS_MODELS_DIR` olarak klonlanır
- `beklenen_eylemler` + `reveal_on` sistemi → lezyon highlight tetikleyici
