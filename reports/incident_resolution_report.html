🔧 Incident Resolution Report

Incident ID: INC-2024-1230-AUTH Date Resolved: 2024-12-30 Severity: Critical (P1) Status: ✅ RESOLVED

1. Issue Summary

Symptom

Users attempting to register via the Next.js frontend received an AxiosError: Network Error when submitting the registration form.

Access to XMLHttpRequest at 'http://localhost:8000/api/auth/register' from origin 'http://localhost:3000' has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header is present on the requested resource.

Impact

FeatureStatus
User Registration❌ Completely blocked
User Login⚠️ Partially affected
Application Usability🔴 Critical

2. Root Cause Analysis

2.1 CORS Misconfiguration (Secondary)

⚠️ Red Herring: The CORS error was a symptom. When FastAPI crashes with 500 error, it fails to send CORS headers.

2.2 passlib/bcrypt Incompatibility (Primary)

The passlib library was failing due to version incompatibility with newer bcrypt package.

File "passlib/handlers/bcrypt.py", line 383 hash = _bcrypt.hashpw(secret, config) TypeError: ...
PackageVersionIssue
passlib1.7.4Outdated bcrypt handler
bcrypt4.1.xBreaking API changes

3. Technical Implementation

3.1 CORS Configuration Update

app/api/main.py

app.add_middleware( CORSMiddleware, allow_origins=[ "http://localhost:3000", "http://127.0.0.1:3000", "http://192.168.1.72:3000", "http://localhost:5173", ], allow_credentials=True, allow_methods=["*"], allow_headers=["*"], )

3.2 Authentication Logic Refactor

app/api/routers/auth.py

❌ Before:

from passlib.context import CryptContext pwd_context = CryptContext(schemes=["bcrypt"]) def get_password_hash(password): return pwd_context.hash(password)

✅ After:

import bcrypt def get_password_hash(password): password_bytes = password.encode('utf-8') salt = bcrypt.gensalt() hashed = bcrypt.hashpw(password_bytes, salt) return hashed.decode('utf-8')

6. Lessons Learned

  1. CORS errors can mask server-side exceptions.
  2. Dependency version pinning is critical.
  3. Direct library usage over wrappers.
  4. Comprehensive error logging is essential.
